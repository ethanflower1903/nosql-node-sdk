Authorization configuration for the driver provided as Config#auth when NoSQLClient instance is created.

Every request to the server made by NoSQLClient methods requires authorization information. The way authorization information is obtained depends on the ServiceType used (set as Config#serviceType):

  • For ServiceType.CLOUD, authorization information is obtained using Oracle Cloud Infrastructure Identity and Access Management (IAM). To enable authorization, you must set IAM configuration in the form of IAMConfig as AuthConfig#iam property.
  • For ServiceType.CLOUDSIM, authorization is not required and you do not need to specify Config#auth property.
  • For ServiceType.KVSTORE, if using secure store, you need to specify the store authentication information such as user name and password as AuthConfig#kvstore property in the form of KVStoreAuthConfig. If using non-secure store, authorization is not required and you do not need to specify Config#auth property.
  • You may also choose to implement your own authorization provider to obtain authorization information for each request depending on the operation performed. In this case, set this provider as AuthConfig#provider property. If the provider is set, ServiceType can be undefined, although it may also be possible to have custom provider for existing service types.

Note that you may specify both AuthConfig#iam and AuthConfig#kvstore in the same configuration object only if Config#serviceType is set to valid ServiceType. If Config#serviceType is not set (in which case the driver will try to deduce service type, see ServiceType), specifying both AuthConfig#iam and AuthConfig#kvstore will result in error.

Type:
  • object
Examples
//This AuthConfig object contains required properties to obtain
authorization from IAM via OCI configuration file
{
    iam: {
        configFile: '~/myapp/.oci/config',
        profileName: 'John'
    }
};
//This AuthConfig object contains required information to connect
//to secure kvstore and assumes the user credentials are stored in
//separate file credentials.json
{
    kvstore: {
        credentials: '/path/to/credentials.json'
    }
}

 If specified, releases any resources associated with this provider when NoSQLClient instance is closed. Note that this operation may be asynchronous in which case this function should return a Promise (resolved value is ignored), otherwise return value is ignored. If error occurs, the function should only log it rather than throwing exception or causing Promise rejection
AuthorizationProvider is an interface to obtain authorization information for NoSQL database operation. By default, the driver will use its own authorization providers based on specified ServiceType and/or presense of service-specific configurations such as IAMConfig and KVStoreAuthConfig. Alternatively, the application may choose to use custom authorization provider and set it as AuthConfig#provider property. This custom provider may be specified either as a getAuthorization function or as an object implementing getAuthorization function.

Source:

Interface to acquire authorization information. Authorization information may be returned as a string or as an object:
  • If represented as a string, it will be used as a value of HTTP Authorization header for the service request.
  • If represented as an object, this object's properties will be added as HTTP headers for the service request.
The specifics depend on the authorization protocol used.
